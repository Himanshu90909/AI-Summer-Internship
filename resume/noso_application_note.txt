ONE AI PROJECT I'VE BUILT:

Automated Email Classifier Using n8n + LLM Workflow Automation

I built an end-to-end AI workflow that automatically classifies incoming emails and routes them to the right action — no human triage needed.

How it works:
1. n8n polls a Gmail inbox at intervals
2. Each email is sent to a FastAPI endpoint I built, which uses structured prompt engineering with the OpenAI API to classify the email into one of 6 categories (Meeting, Leave Request, Support Ticket, Billing, Project Update, Other) — with a confidence score, priority level, and extracted entities (dates, names, amounts)
3. Based on the classification, n8n's 14-node workflow with Switch routing triggers category-specific actions:
   - Meeting → auto-creates a Google Calendar event
   - Leave Request → notifies HR via Slack
   - Support Ticket → alerts the support team channel in Slack
   - Billing/Invoice → forwards to the finance team
   - Project Update → labels and archives in Gmail

What I learned:
- Prompt engineering for reliable structured output (JSON classification with confidence scores) — the LLM needs to be grounded in a strict schema, not free-form
- Building evaluation feedback loops — I tested classification accuracy across edge cases (ambiguous subject lines, multi-topic emails) and iterated on the prompt to improve reliability
- Agent-style routing — the classification triggers downstream actions, which is fundamentally an agentic workflow pattern
